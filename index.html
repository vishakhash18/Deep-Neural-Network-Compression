Fruits-360 Model Compression Pipeline

End-to-End Evaluation of PyTorch Neural Network Compression

Overview

This repository provides a complete pipeline for training and compressing a custom multimodal PyTorch Deep Neural Network (DNN). It focuses on the Fruits-360 dataset, which is automatically downloaded via the kagglehub library.

The core objective is to demonstrate Model Compression—specifically Pruning and Quantization—and compare the memory footprint and accuracy of the base model against its compressed variants.

Core Concepts Used

Multimodal Feature Extraction

The model (FruitModel) does not just take raw pixels. It processes 4 distinct types of inputs generated by the data loader:

  • LBP (Local Binary Patterns): Extracts texture information.
  • Canny Edges: Extracts structural and shape outlines.
  • Color Features: Provides color histograms or mean color values.
  • Shape Features: Geometric properties of the fruits.

Model Pruning

Removes redundant or less important weights from the neural network. The implementation uses magnitude-based pruning where weights below a certain threshold (t=0.05) are zeroed out using a binary mask. This increases model sparsity.

Model Quantization

Reduces the precision of the weights. The custom layers cluster the weights into k=16 distinct values. This effectively drops the representation from 32-bit floats to ~4-bit quantized representations, heavily compressing the model size.

Libraries & Dependencies

  • PyTorch (torch, torch.nn): The primary deep learning framework used for defining the model layers, training loops, and performing tensor operations. Includes support for hardware acceleration (CUDA/MPS).
  • KaggleHub (kagglehub): Used in main.py to seamlessly and programmatically download the "Fruits-360" dataset directly from Kaggle without manual intervention.
  • NumPy / OS / Math: Built-in and standard numerical libraries used for calculating metrics, file sizes, quantizing math functions, and saving the final compressed model as a sparse numpy array (.npz).

How to Run / Access / Use

  1. Prerequisites: Ensure you have Python installed along with PyTorch and KaggleHub. 
    pip install torch torchvision kagglehub numpy
  2. Execution: Simply run the main.py file from the base directory. 
    python main.py

What happens when you run it?

  1. The script will download the Fruits-360 dataset.
  2. Extract features and prepare PyTorch DataLoaders.
  3. Initialize the FruitModel and move it to the available hardware accelerator.
  4. Train the Baseline Model for 10 epochs and save it to output_models/init.pth.
  5. Apply Pruning (threshold 0.05), check sparsity, fine-tune for 5 epochs, and save to output_models/pruned_net.pth.
  6. Apply Quantization (k=16), fine-tune for 5 epochs, and save to output_models/quantized_net.pth.
  7. Save the final heavily compressed model to output_models/final_compressed.npz.
  8. Output a comparison report of the file sizes (in MB) and percentage reduction compared to the baseline.
